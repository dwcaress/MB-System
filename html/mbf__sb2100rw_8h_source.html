32 /*
33  * Notes on the MBF_SB2100RW data format:
34  * 1. SeaBeam 1000/2100 multibeam systems output raw data in an
35  * ascii format. The data consists of a number of different
36  * multi-line ascii records.
37  * 2. The 2100/2100 systems output up to 151 beams of bathymetry
38  * and 2000 pixels of sidescan measurements, along with a plethora
39  * of other information.
40  * 3. The records all include navigation and time stamp information.
41  * The record types are:
42  * PR: sonar parameter record (roll bias, pitch bias, sound velocity profile)
43  * TR: sonar text record (comments)
44  * SB: sub-bottom data record (undefined as yet)
45  * DR: bathymetry data record (bathymetry and per-beam amplitudes)
46  * SS: side scan data record
47  * 4. A single ping usually results in both DR and SS records. The PR record
48  * occurs every 30 minutes or when the sound velocity profile is changed.
49  * 5. The kind value in the mbsys_sb2k_struct indicates whether the
50  * mbsys_sb2k_data_struct structure holds data from a ping or
51  * data from some other record:
52  * kind = 1 : data from a ping
53  * (DR + SS)
54  * kind = 2 : comment (TR)
55  * kind = 8 : sonar parameter (PR)
56  * 6. The data structure defined below includes all of the values
57  * which are passed in SeaBeam 1000/2100 records.
58  */
