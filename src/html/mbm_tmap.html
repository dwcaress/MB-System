NAME


mbm_tmap - Macro to convert XYZ point clouds to 3D Tiles and generate a Cesium viewer.

 

VERSION


Version 5.0

 

SYNOPSIS


mbm_tmap -Ipointcloud.xyz
[-Otiles_dir]
[-Pport]
[--serve-only | --no-convert]
[-N]
[-S]
[-W]
[-V]
[-H]

 

DESCRIPTION


mbm_tmap is a macro that automates point cloud web visualization setup.
It runs py3dtiles convert on an input XYZ point cloud file, writes an
index.html Cesium viewer file into the output tiles directory, and can
optionally start a local HTTP server using python -m http.server.

By default mbm_tmap performs conversion from XYZ input. If
--serve-only (or --no-convert) is specified, conversion is skipped
and an existing tiles directory is served directly.

This macro is intended for quick local testing and browsing of tiled point cloud
data products.

OPTIONS





-I



pointcloud.xyz



Input XYZ point cloud file. This option is required unless
--serve-only or --no-convert is specified.


-O



tiles_dir



Output directory for 3D Tiles and generated index.html.
Default is tiles.


-P



port



HTTP server port used when starting python -m http.server.
Default is 8000.


--serve-only, --no-convert






Skip py3dtiles conversion and serve an existing tiles directory.
In this mode -O should point to an existing tiles directory.
If index.html already exists it is reused.


-N






Do not open a browser automatically.


-S






Do not start a local HTTP server.


-W






Wait on the server process after starting it. Use Ctrl+C to stop.


-V






Verbose output.


-H






Print help message and exit.



 

EXAMPLES


Convert an XYZ file to tiles and start local server:



mbm_tmap -I cloud.xyz -O tiles -V

Serve an existing tiles directory without reconversion:



mbm_tmap --serve-only -O tiles -P 8081 -W -V

Convert an XYZ file to tiles without starting server:



mbm_tmap -I cloud.xyz -O tiles -S

Start server and keep the macro attached to server lifetime:



mbm_tmap -I cloud.xyz -O tiles -W
