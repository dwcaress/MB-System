NAME


mbmakedatalist - Macro to generate a datalist of the swath
files in a specified directory.


 

VERSION


Version 5.0


 

SYNOPSIS


mbmakedatalist
[-Bsize|--min-size=size]
[-Fformat|--format=format]
[-Idirectory|--input=directory]
[-L|--skip-latest]
[-Odatalist|--output=datalist]
[-P|--ignore-processed]
[-Ssuffix|--suffix=suffix]
[-T|--no-time-sort]
[-H|--help]
[-V|--verbose]


 

DESCRIPTION


Mbmakedatalist is a macro to generate an MB-System datalist file
referencing all identifiable swath files in the specified target directory.


Datalists are fundamental structures in MB-System workflows because they
allow programs to operate on sets of swath data files.
Datalist files are text lists of swath data files and their format ids with each
file entry taking up a single line. These lists may contain references to other
datalists, making them recursive. Datalists may also contain comments and parsing
directives that, for example, determine whether parsing returns references to
raw or processed data files. See the MB-System manual page for details
on the format and structure of datalists.


This macro is used to easily construct a datalist referencing swath files in a
directory, often as part of setting up the MB-System processing environment
for a new seafloor mapping dataset. By default, mbmakedatalist tests
all of the filenames in the local directory for adherence to the MB-System
swath file naming convention (see the mbsystem manual page for details),
and makes an output datalist including all of the files that can be recognized
as swath data. This can include files with suffixes such as ".mb88", that are
recognized as format 88 (Reson s7k) files, and files with recognized vendor
format suffixes, such as "_raw.all" or ".all" for old and recent Kongsberg
multibeam data, respectively. The suffix of the files to be considered can be
specified with the -Ssuffix (or --suffix=suffix) option;
under most circumstances this is the best approach as it prevents
mbmakedatalist from including files that are not intended for inclusion
by the user yet satisfy one of the many recognized naming conventions.
By default, mbmakedatalist infers the format id of included files using
the naming conventions. The format id can be specified using the
-Fformat (or --format=format) option.


MB-System differentiates between raw and processed files by inserting the
letter "p" into the processed file name immediately before the filename suffix.
For instance, for Reson 7k data the raw files will have file suffix of *.mb88,
and the processed files generated by mbprocess will end with *p.mb88.
When using mbmakedatalist to generate a datalist by specifying a suffix,
processed files can be excluded from the datalist by also specifying the -P
(or --ignore-processed) option. Thus it is possible to generate a datalist
of raw files in a directory containing both raw and processed file.


The output datalist will be placed in the current working
directory. The output datalist is named datalist.mb-1
by default, but can be specified with the -O (or --output) option.
If no target directory is specified with the -I (or --input) option,
then the current working directory is used. In order for swath files to be
recognized and included in the datalist, they must conform to the MB-System
swath file naming convention (see the mbsystem manual page for details).


Most seafloor mapping sonars and datalogging packages produce original swath
data files with names constructed so that standard directory listings are in
time order. However, Kongsberg multibeam data filenames usually start with a survey
file number that restarts at 0001 whenever datalogging is stopped and restarted.
Consequently, when data from multiple instances of logging are present in a
single directory, the simple listing of files will not in general be in time
order. If the file suffix is declared to be ".all", ".ALL", ".kmall", or ".KMALL"
using the -S (or --suffix) option, then mbmakedatalist will
attempt to sort the swath files into time order before generating the final
datalist. This behavior can be disabled by specifying the -T
(or --no-time-sort) option. Kongsberg dataloggers often create junk data
files with names such as "9999.all" or "9999.ALL"; these filenames are ignored.

OPTIONS


Each option has both a traditional single-letter form and a GNU-style long form;
the two forms are fully equivalent and may be used interchangeably.



-B   size ,  --min-size=size



Sets a size threshold in kilobytes below which a file will be ignored and not
included in the output datalist file.

-F   format ,  --format=format



By default, mbmakedatalist infers the format id of included files using
the naming conventions. This option specifies the format id that is assigned to
all files.

-H ,  --help



Causes the program to print out a description of its operation and then exit
immediately.

-I   directory ,  --input=directory



Sets the path of the target directory to scan for swath files.

-L ,  --skip-latest



Causes the last file in the directory listing to be omitted from the output
datalist. The intent of this option is to exclude the most recent file in an
actively data logging context, since the most recent file is still being logged.
This presumes that filenames sort in time order, an assumption that does not hold
for all file naming conditions or if the -T (--no-time-sort)
option has been used to disable time ordering.

-O   datalist ,  --output=datalist



Sets the name of the output datalist file. Default: datalist.mb-1

-P ,  --ignore-processed



Suppresses the inclusion of processed files in the output datalist (e.g.
*p.mb88).

-S   suffix ,  --suffix=suffix



Sets the filename suffix of swath files considered for inclusion in the output
datalist. Under most circumstances using this option explicitly is the best
approach as it prevents mbmakedatalist from including files that are not
intended for inclusion by the user yet satisfy one of the many recognized naming
conventions.
If the file suffix declared with this option corresponds to one of the formats
associated with Kongsberg multibeam sonar data, then mbmakedatalist will
attempt to sort the swath files into time order before generating the final
datalist. This behavior applies to files with the ".all" or ".ALL" suffix used
for raw logged Kongsberg files, and also to files with suffixes following the
MB-System convention (e.g. ".mb56", ".mb57", ".mb58", ".mb59" suffixes).
This behavior is disabled by the -T (--no-time-sort) option.

-T ,  --no-time-sort



By default, when the -S (--suffix) option is used to specify a file
suffix corresponding to Kongsberg multibeam data, mbmakedatalist attempts
to sort the data files into time order before outputting the datalist. This option
disables that behavior.

-V ,  --verbose



Causes mbmakedatalist to print out status messages.




 

EXAMPLES


Suppose we have a set of swath files in a directory and we
want to construct a datalist referring to those files. A simple
directory listing shows four files:




        20051019_154030.mb88


        20051019_162540.mb88


        20051019_171756.mb88


        junk.txt




of which three are identifiable as swath files because of the
".mb88" file suffix. Running mbmakedatalist in this
directory with no arguments other than verbosity:




        mbmakedatalist -V




or equivalently using the long form:




        mbmakedatalist --verbose




produces a datalist file named datalist.mb-1. The contents of
this file are:




        20051019_154030.mb88 88


        20051019_162540.mb88 88


        20051019_171756.mb88 88




where the file junk.txt has been ignored.


Suppose instead the data are Kongsberg .all files spread across
multiple logging sessions in a subdirectory called rawdata,
and we want only raw (unprocessed) files, a datalist named
kongsberg.mb-1, and we wish to exclude the file still being
actively logged. The following two commands are equivalent:




        mbmakedatalist -I rawdata -S .all -P -L -O kongsberg.mb-1





        mbmakedatalist --input=rawdata --suffix=.all \


            --ignore-processed --skip-latest --output=kongsberg.mb-1




In both cases mbmakedatalist will automatically sort the
Kongsberg files into time order before writing the datalist.
