24 /*
25  * mbf_em12ifrm.h defines the data structures used by MBIO functions
26  * to store multibeam data read from the MBF_EM12IFRM format (MBIO id 58).
27  *
28  * Author: D. W. Caress
29  * Date: December 5, 2000
30  *
31  */
32 /*
33  * Notes on the MBF_EM12IFRM data format:
34  * 1. IFREMER uses their own software to process swath data.
35  * Currently (2004) the software is called CARAIBES, but
36  * during the 1990's they used an earlier package called
37  * TRISMUS to process multibeam data. This format was used
38  * and generated as part of TRISMUS processing of EM12S and
39  * EM12D multibeam data. TRISMUS recast the data as parallel
40  * files for bathymetry (.SO suffix), imagery (.IM suffix),
41  * and asynchronous navigation (.NA suffix). We recommend
42  * that users translate the TRISMUS data to the current
43  * mbsystem format for processing Simrad multibeam data
44  * (EM300MBA - format 57) before processing.
45  * 2. This format is supported read-only in MB-System.
46  * 3. The systems of interest:
47  * EM-12S: Deep water 12kHz multibeam sonar with up to
48  * 81 beams of bathymetry and up to 523 sidescan
49  * samples per bathymetry beam.
50  * EM-12D: Deep water 12kHz multibeam sonar with up to
51  * 81 beams of bathymetry and up to 523 sidescan
52  * samples per bathymetry beam. This is a dual
53  * system which produces alternating port and
54  * starboard pings to provide a 150 degree swath.
55  * 4. The navigation data contain NMEA type ascii strings
56  * beginning with "$CASTM". The sources of these strings
57  * include "NACOU", "NAGP1", and "NAGP2".
58  * MB-System uses only the "NACOU" strings.
59  * 5. Each bathymetry record is 1032 bytes long and begins
60  * with a 35 character string containing the identifier
61  * and time stamp. The first six characters are one of
62  * the following:
63  * "$12SOC": EM12S ping
64  * "$12SOB": EM12D port ping
65  * "$12SOT": EM12S starboard ping
66  * "$COMM:": Comment (MB-System only)
67  * 6. The imagery records are of variable length and begin
68  * with a 35 character string containing the identifier
69  * and time stamp. The first six characters are one of
70  * the following:
71  * "$12IMC": EM12S ping
72  * "$12IMB": EM12D port ping
73  * "$12IMT": EM12S starboard ping
74  * As with the Simrad vendor formats, multiple sidescan
75  * datagrams are recorded for each ping.
76  *
77  */
